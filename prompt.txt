You are an expert full-stack developer, Chrome Extension engineer, game logic engineer, and TypeScript architect.

Build a complete, working, free-stack MVP for a multiplayer Chrome Extension game called “Movie Guess Battle”.

The game is inspired by the Indian paper movie guessing game where a movie name is shown with blanks, for example:

D _ _ _ M

Players guess alphabets one by one. If the guessed alphabet is correct, it appears in the correct positions. If the guessed alphabet is wrong, one letter is deducted from a life word such as BOLLYWOOD or HOLLYWOOD. When all letters from the life word are exhausted, the guessing side loses the round.

Game flow:

1. User opens the Chrome Extension popup.
2. Popup shows:

   * Create Room
   * Join Room
   * Last Room
   * How to Play
3. Create Room opens the full game page.
4. Host enters player name.
5. Host creates a room.
6. Generate a short room code like AX72K.
7. Host can copy invite code or invite link.
8. Other players join using room code or invite link.
9. Lobby shows:

   * Room code
   * Player list
   * Host badge
   * Connection status
   * Category: Bollywood, Hollywood, Mixed
   * Difficulty: Easy, Medium, Hard, Expert
   * Life word: BOLLYWOOD or HOLLYWOOD
   * Hint settings
   * Start setup button
10. Host starts round setup.
11. System selects a movie based on category and difficulty.
12. Host sees the Host Hint Setup screen.
13. Host chooses hint mode:

* Manual hints
* Random hints
* Smart random hints
* No hints
* Difficulty-based auto hints

14. Host previews masked movie.
15. Host locks hints and starts round.
16. Players only see the masked movie, not the full movie.
17. Players guess alphabets turn by turn.
18. Correct guess reveals letters and adds score.
19. Wrong guess deducts one letter from the life word.
20. Full movie guess is allowed.
21. Round ends when movie is solved or life word is exhausted.
22. Result screen shows:

* Correct movie
* Winner/loser
* Scoreboard
* Next round button for host

Player rules:

* Minimum players: 2
* Recommended players: 3 to 4
* Maximum players: 4
* Exactly one host per room
* Only host can change room settings
* Only host can start round
* Only host can end round
* Only host can start next round
* If host leaves, assign host role to the next active player
* A player cannot guess the same alphabet twice
* A player cannot guess when it is not their turn
* Spaces, punctuation, and numbers should always be visible
* Alphabet guesses should be case-insensitive
* Movie completion should ignore spaces, punctuation, and case
* Repeated letters should reveal together when guessed by a player
* Room should handle refresh/reconnect using stored player id and room code

Scoring:

* Correct alphabet guess: +10
* Wrong alphabet guess: 0 by default
* Optional wrong penalty setting: -5
* Correct full movie guess: +50
* Wrong full movie guess: deduct 2 life letters
* Skipped turn: 0
* Timer expired: skip turn

Turn timer:

* Default: 30 seconds
* Host can configure 15, 30, 45, or 60 seconds
* If timer expires, auto-skip and move to next player
* Broadcast timer changes to all players
* Do not rely only on client-side timer for final state; database state should remain source of truth

Life word logic:

* Life word can be BOLLYWOOD or HOLLYWOOD
* Show all life word letters at round start
* On wrong alphabet guess, remove/hide one letter from the end
* On wrong full movie guess, remove/hide two letters
* When life remaining reaches 0, round is lost
* Add smooth animation for deducted letters

Advanced Hint Letter Placement System:
Before each round starts, the host can control the hint letters shown in the movie mask.

Hint modes:

1. Manual Hint Mode
2. Random Hint Mode
3. Smart Random Hint Mode
4. No Hint Mode
5. Difficulty-Based Auto Hint Mode

Manual Hint Mode:

* Host sees the full movie title during setup.
* Host can click exact character positions to reveal as hints.
* Only alphabet positions are clickable.
* Spaces, punctuation, and numbers are always visible and not counted as hints.
* Host cannot reveal every letter.
* Enforce that at least 40% of alphabet letters remain hidden.
* Add a setting called “Reveal duplicate letters automatically”.
* If duplicate reveal is enabled and host selects O in DHOOM, both O letters can reveal.
* If duplicate reveal is disabled, only the selected O position reveals.
* Host can clear selected hints.
* Host can preview the final masked movie before starting.

Random Hint Mode:

* System randomly selects hint positions.
* Randomization should only select alphabet characters.
* Randomization should not select spaces, punctuation, or numbers.
* Add “Randomize Again” button.
* Host can preview result before starting.
* Randomization should respect difficulty limits.
* Randomization should not make the movie too easy.

Smart Random Hint Mode:

* Reveal hints more intelligently.
* Avoid revealing all vowels.
* Avoid revealing only first and last letters every time.
* Avoid revealing letters that make short titles too obvious.
* For short movies, reveal fewer hints.
* For long movies, reveal slightly more hints.
* Respect difficulty-based max hints.
* Avoid predictable patterns.

No Hint Mode:

* Do not reveal alphabet letters.
* Only spaces, punctuation, and numbers remain visible.
* Example:
  3 IDIOTS becomes 3 _ _ _ _ _ _

Difficulty-Based Auto Hint Mode:
Easy:

* Reveal first and last alphabet letter when allowed.
* Reveal 25% to 30% of alphabet positions.
* Allow vowels by default.

Medium:

* Reveal 15% to 20% of alphabet positions.
* Avoid too many vowels.
* First and last letters optional.

Hard:

* Reveal 5% to 10% of alphabet positions.
* Do not automatically reveal first and last letters.
* Prefer internal letters.

Expert:

* Reveal 0% to 5% of alphabet positions.
* Mostly no hints.

Hint validation:

* Do not allow hint positions outside title length.
* Do not allow hint positions on spaces.
* Do not allow hint positions on punctuation.
* Do not allow hint positions on numbers.
* Do not allow all letters to be revealed.
* Enforce minimum hidden percent.

* If allowFirstLetterHint is false, first alphabet position cannot be selected.
* If allowLastLetterHint is false, last alphabet position cannot be selected.

* Players can read public room state for rooms they joined.

Important security limitation note:
